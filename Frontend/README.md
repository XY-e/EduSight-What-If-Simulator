## Backend API Response Specification

The frontend dashboard is designed to consume data directly from the backend APIs. To ensure seamless integration between the frontend, ML Engine, Risk Engine, and Recommendation Engine, the following response structures are expected.

### GET /students/:id

This endpoint should return the student's current profile and baseline information.

```json
{
  "id": "STU-001",
  "name": "Muhammad Ali bin Faisal",
  "form": "Form 4 Jujur",
  "school": "SMK Damansara Utama",
  "currentScore": 89,
  "currentRisk": "High Risk",
  "attendance": 62,
  "grades": 60,
  "counselling": 0,
  "welfare": 0
}
```

### POST /simulate

The frontend will send the following payload:

```json
{
  "studentId": "STU-001",
  "inputs": {
    "attendance": 80,
    "grades": 70,
    "counselling": 3,
    "welfare": 1
  }
}
```

The backend should process the simulation using the ML Engine, Risk Engine, and Recommendation Engine, then return the following structure:

```json
{
  "baselineScore": 89,
  "projectedScore": 82,
  "riskLevel": "High Dropout Risk",
  "riskLabel": "high risk",
  "scoreChangeText": "↓ 7 pts",
  "dropoutProbability": 90,
  "insightText": "Simulation completed successfully.",
  "narrative": "At the ongoing trajectory, Muhammad Ali is at high risk of dropping out within 3 months (90% probability). Immediate action is recommended. Simulated interventions could reduce the risk score by 7 points, bringing 3-month dropout probability down to 82%.",
  "weights": {
    "attendance": 45,
    "academic": 29,
    "socioeconomic": 13,
    "family": 12
  },
  "recommendations": [
    {
      "category": "attendance",
      "urgency": "critical",
      "title": "Attendance Improvement Plan (AIP)",
      "description": "Worsening attendance trend detected. Create a formal Attendance Improvement Plan with weekly check-ins.",
      "expected_impact": "stabilise trend, reduce risk by ~5 pts"
    }
  ]
}
```

### Notes

* `baselineScore` represents the student's current risk score.
* `projectedScore` represents the simulated risk score after applying interventions.
* `dropoutProbability` should be returned as a percentage value (0–100).
* `narrative` should be generated by the backend narrative builder and displayed directly in the probability section.
* `weights` should contain the risk factor importance values calculated by the Risk Engine.
* `recommendations` should be generated by the Recommendation Engine and sorted by urgency.
* The frontend will display the top three recommendations based on urgency priority:

  * Critical
  * High
  * Medium
  * Routine
