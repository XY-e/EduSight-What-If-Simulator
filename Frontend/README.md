## Backend API Response Specification

### GET /students/:id

This endpoint should return the student's profile information and current risk status.

```json
{
  "id": "string",
  "name": "string",
  "form": "string",
  "school": "string",
  "currentScore": "number",
  "currentRisk": "string",
  "attendance": "number",
  "grades": "number",
  "counselling": "number",
  "welfare": "number"
}
```

---

### POST /simulate

#### Request Body

```json
{
  "studentId": "string",
  "inputs": {
    "attendance": "number",
    "grades": "number",
    "counselling": "number",
    "welfare": "number"
  }
}
```

#### Response Body

The backend should return the simulation result generated by the ML Engine, Risk Engine, and Recommendation Engine.

```json
{
  "baselineScore": "number",
  "projectedScore": "number",
  "riskLevel": "string",
  "riskLabel": "string",
  "scoreChangeText": "string",
  "dropoutProbability": "number",
  "insightText": "string",
  "narrative": "string",
  "weights": {
    "attendance": "number",
    "academic": "number",
    "socioeconomic": "number",
    "family": "number"
  },
  "recommendations": [
    {
      "category": "string",
      "urgency": "critical | high | medium | routine",
      "title": "string",
      "description": "string",
      "expected_impact": "string"
    }
  ]
}
```

### Notes

* `baselineScore` represents the student's current risk score.
* `projectedScore` represents the simulated risk score after applying the selected interventions.
* `dropoutProbability` should be returned as a percentage value.
* `narrative` will be displayed in the 3-Month Dropout Probability section.
* `weights` will be displayed in the Simulated Risk Factor Weights panel.
* `recommendations` will be displayed in the Recommendations panel.
* The frontend will automatically sort recommendations by urgency and display the top three recommendations.
